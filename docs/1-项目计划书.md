# 南锣鼓巷旅游导览系统 - 项目计划书

## 一、胡同选择理由

### 1.1 选择南锣鼓巷的原因

南锣鼓巷是北京最具代表性的历史文化街区之一，选择该胡同作为导览系统的目标区域，基于以下考量：

**历史文化价值**
- 南锣鼓巷始建于元代，距今已有740余年历史，是北京最古老的街区之一
- 保存了大量明清时期的传统四合院建筑，是研究北京胡同文化的活化石
- 曾居住过茅盾、齐白石、蒋介石等众多历史名人，文化底蕴深厚

**旅游资源丰富**
- 全长787米，东西两侧各有8条胡同整齐排列，形成"鱼骨状"格局
- 拥有名人故居、历史建筑、特色美食、文创小店等多种类型景点
- 年接待游客量超过千万人次，是北京最热门的旅游目的地之一

**导览需求明确**
- 景点分布密集但信息分散，游客难以系统了解
- 缺乏统一的数字化导览平台，存在信息获取不便的痛点
- 适合开发集信息展示、路线规划于一体的导览系统

## 二、核心功能列表

### 2.1 功能模块概览

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| 胡同概览 | 展示南锣鼓巷历史、地理位置、特色简介 | P0 |
| 景点浏览 | 展示10+个景点的列表和详细信息 | P0 |
| 景点搜索 | 支持按名称、描述关键词搜索景点 | P0 |
| 推荐路线 | 提供2条主题游览路线及景点顺序 | P0 |
| 后台管理 | 管理员可维护景点、路线数据 | P1 |

### 2.2 功能详细说明

**胡同概览**：首页展示南锣鼓巷的历史沿革、地理位置、全长信息及特色介绍，帮助游客快速了解目的地。

**景点浏览**：以卡片形式展示景点列表，包含图片、名称、类型标签、简介；点击可查看详情页，包含图片轮播、详细介绍（≥100字）、开放时间、游览提示等。

**景点搜索**：在景点列表页提供搜索框，支持中文关键词模糊匹配，实时显示搜索结果。

**推荐路线**：提供"经典文化游"和"休闲美食游"两条主题路线，展示游览时长、距离、包含景点，支持展开查看景点顺序。

## 三、技术选型说明

### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端展示层                           │
│  ┌─────────────────────┐    ┌─────────────────────┐        │
│  │   Vue3 + Vite       │    │   微信小程序         │        │
│  │   Element Plus      │    │   原生开发           │        │
│  └─────────────────────┘    └─────────────────────┘        │
├─────────────────────────────────────────────────────────────┤
│                        后端服务层                           │
│  ┌─────────────────────────────────────────────────┐       │
│  │   Django 4.2 + Django REST Framework            │       │
│  │   RESTful API 设计                              │       │
│  └─────────────────────────────────────────────────┘       │
├─────────────────────────────────────────────────────────────┤
│                        数据存储层                           │
│  ┌─────────────────────────────────────────────────┐       │
│  │   SQLite 数据库                                 │       │
│  └─────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| Django | Python生态成熟，自带Admin后台，ORM简化数据库操作，开发效率高 |
| DRF | 快速构建RESTful API，序列化器简化数据处理，文档完善 |
| SQLite | 轻量级数据库，无需额外安装，适合中小型项目，便于部署 |
| Vue3 | 组合式API提升代码复用性，响应式系统性能优秀，生态丰富 |
| Vite | 开发服务器启动快，热更新即时，构建速度优于Webpack |
| Element Plus | Vue3官方推荐UI库，组件丰富，文档完善，中文支持好 |
| 微信小程序 | 用户基数大，无需下载安装，适合移动端轻量使用场景 |
